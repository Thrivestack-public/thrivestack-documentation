User & Account Events

Install the SDK, identify users, and track every auth lifecycle event.

Events instrumented

identify group signed_up signed_in signed_out account_created account_added_user invite_sent
0

Run this in repos that contain

  • User signup or registration handlers
  • Login and logout handlers
  • Account, organization, or workspace creation logic
  • User invite flows
~/acme — repositories
auth-service signup · login · logout
admin-portal accounts · invites
api-gateway user identity
billing-service
notification-service
data-pipeline
Auth in a separate repo? Run this prompt in the repo where your signup and login handlers live. If your main app also reads userId or groupId, run it there too.
1

The AI will ask you

  • Is this a B2B app (accounts/orgs) or B2C (individual users)?
  • Do you have European customers? (affects PII handling)
  • Do you want to check for disposable or abusive email addresses during signup?
2

Read https://docs.thrivestack.ai/ai-setup/product/scoped/auth/llms.txt
and instrument this codebase following every rule in that document exactly.
