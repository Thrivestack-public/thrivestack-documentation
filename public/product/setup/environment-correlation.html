Environment Setup for Correlation

ThriveStack correlates marketing and product signals using a shared API key environment. Using the wrong environment on your live site will silently break attribution — marketing visits and product events will not be joined. This page explains the correct setup for each intelligence type.

Important: Correlation between Marketing Intelligence and Product Intelligence only works when both use an API key from the same environment. A development key on one side and a production key on the other will break attribution.

Marketing Intelligence

ThriveStack's Marketing Intelligence script captures visitor attribution data — UTM parameters, referrer, landing pages — and links them to signed-up users and accounts. For this correlation to work end-to-end, the API key used on your marketing site must match the environment used in your product.

Rule

Where API Key to use
Local / staging / testing Development environment API key
Live production site Production environment API key — required for correlation

What to do

  1. Testing locally? Use your Development environment API key. This keeps test traffic isolated from real data.
  2. Pushing to your live marketing site? Switch to your Production environment API key before deploying. This is required for ThriveStack to join marketing visits with product sign-ups and usage.
  3. No Production environment yet? Click the environment dropdown in the top-left of the ThriveStack setup page and select New environment to create one and get its API key.
Note: Your marketing site and product app must both use API keys from the same environment for correlation to work. If your product runs on Production, your marketing site must also use a Production key.

Product Intelligence

ThriveStack's Product Intelligence script tracks user and account activity inside your app — sign-ups, feature usage, activation milestones. For marketing attribution to flow through to product events, your app must use an API key from the same environment as your marketing site.

Rule

Where API Key to use
Local / staging / testing Development environment API key
Live production app Production environment API key — required for correlation

What to do

  1. Testing locally? Use your Development environment API key. This keeps test events isolated from real production data.
  2. Pushing to your live product app? Switch to your Production environment API key before deploying. This is required for ThriveStack to correlate marketing attribution with product sign-ups and feature usage.
  3. No Production environment yet? Click the environment dropdown in the top-left of the ThriveStack setup page and select New environment to create one and get its API key.
Note: Your product app and marketing site must both use API keys from the same environment for correlation to work. If your marketing site runs on Production, your product app must also use a Production key.
