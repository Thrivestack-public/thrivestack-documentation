ThriveStack → HubSpot

Push MRR, activation status and engagement signals from ThriveStack into HubSpot company properties — so your sales and CS teams always have live product data inside HubSpot.

What you need

A HubSpot account — Free tier or above Super Admin or CRM Admin access in HubSpot to authorize write permissions ThriveStack product telemetry set up so account-level signals are available

What gets exported

ThriveStack writes to HubSpot Company properties. On the first sync it creates a ThriveStack property group in your portal — you'll see it appear under any company record and can use it in HubSpot workflows, lists and sequences.

Use in HubSpot workflows: Once properties are synced you can build automations — for example, enroll accounts into an upsell sequence when ts_activation_status = complete and ts_mrr = 0.

How to set up

1
Open the CRM sync wizard /analyze/onboarding/crm

Go to Analyze → Onboarding → CRM and click Add Connection.

2
Select "Outbound" direction

On the direction step choose ThriveStack → HubSpot.

3
Authorize with HubSpot

Click HubSpot then Authorize with HubSpot. Sign in with a Super Admin or CRM Admin account — write access is required to create the ThriveStack property group.

4
Map fields

Select which ThriveStack fields to push. You can push a subset — for example, just MRR and activation status if that's all your sales team needs. All selected fields map to standard ts_-prefixed company properties in HubSpot.

5
Set a first-sync filter (optional)

Use first-sync rules to control which accounts get pushed — for example, only accounts with MRR > 0 or activation status = complete. This prevents cluttering HubSpot with trial or empty accounts.

6
Review and activate

Click Create Connection. ThriveStack creates the property group in HubSpot and runs the first sync. Properties will appear under ThriveStack on each matched company record.

How accounts are matched

ThriveStack matches your accounts to HubSpot companies using the domain field. On the first sync it writes a ts_id property to each matched company. All subsequent syncs use this stable identifier — so if a company's domain changes in HubSpot, the sync still works correctly.

If no match is found by domain, ThriveStack creates a new company record in HubSpot using your account's name and domain.

Sync schedule

RunModeNotes
Daily at 03:00 UTC Incremental Runs 1 hour after inbound sync (if both directions are active) so the freshest ThriveStack data is pushed. Only accounts with changed values since the last run are updated in HubSpot.
Manual trigger Incremental Click ⋯ → Trigger Sync on your connection card to push the latest values immediately — useful after a large activation campaign or MRR update you want reflected in HubSpot right away.

Troubleshooting

ThriveStack properties not appearing in HubSpot

Properties are created on the first outbound sync. If you can't see them, go to HubSpot → Settings → Properties and filter by group ThriveStack. If the group doesn't exist, go back to ThriveStack and use ⋯ → Trigger Sync to force a retry.

Some accounts aren't being pushed

Check your first-sync filter rules — accounts that don't match the filter criteria are skipped. You can view and edit filters from ⋯ → Edit on your connection card.

Property values look stale in HubSpot

The outbound sync runs once daily. For the most current values use ⋯ → Trigger Sync to push immediately. If values are still wrong after a manual trigger, verify that ThriveStack is receiving telemetry events — go to Analyze → Accounts and check the account's engagement and revenue data directly.

If this doesn't resolve your issue or you're running into a problem we haven't covered, email us at support@thrivestack.ai and we'll help you.
